Frequently Asked Questions

Everything you need to know about 2ndCTO

What is Bus Factor? +
Bus Factor is the number of people that can be "hit by a bus" (leave the project) before it fails. A bus factor of 1 means one person holds critical knowledge. We analyze git history to calculate this and identify knowledge silos.
Is my code safe with 2ndCTO? +
Absolutely. We never store your actual code. We only analyze metadata: file paths, git history, and dependency manifests. For public repos, we don't even need permissions. Your intellectual property stays on your servers.
How accurate is the risk score? +
Our risk score combines multiple factors: security findings (weighted by severity), bus factor, code complexity, and dependency freshness. It's calibrated against known vulnerable repositories and continuously improved.
What languages are supported? +
Currently: JavaScript, TypeScript, Python, Go, Java, Ruby, PHP, Rust, and C/C++. Security scanning works for all languages. Bus factor analysis is language-agnostic (uses git history). We're adding more languages monthly.
Can I use this for private repositories? +
Yes! Connect your GitHub account and analyze private repos. We use OAuth with minimal permissions (read-only access). You can revoke access anytime. We never store your code or tokens longer than necessary.
What's the difference from Snyk or SonarQube? +
Snyk and SonarQube focus on security and code quality. 2ndCTO focuses on team resilience: bus factor, knowledge distribution, and legacy modernization. We complement these tools, not replace them. Plus, we're open source.
Is this free? +
Public repositories: Completely free, unlimited analyses. Private repositories: Free tier includes 5 repos. Pro tier coming soon with unlimited private repos, team collaboration, and advanced reports.
How do I modernize legacy code? +
Coming in v1.0! Our modernization engine will convert ES5 ‚Üí ES2022, callbacks ‚Üí async/await, Python 2 ‚Üí 3, and more. You'll get a diff view showing before/after with automated tests to validate correctness.
